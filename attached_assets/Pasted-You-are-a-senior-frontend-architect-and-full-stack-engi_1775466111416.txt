You are a senior frontend architect and full-stack engineer.

Build a production-grade interactive ER Diagram Canvas application that visualizes database schema from a JSON configuration file. The system must support dynamic filtering, table selection, and relationship visualization.

---

# 🎯 GOAL

Create a web-based ER Diagram tool where:

* Users can visualize database schema (tables + relationships)
* Select specific tables/modules
* Filter relationships dynamically
* Render diagrams in an interactive canvas

---

# 🧱 INPUT DATA

The application will consume a JSON schema with:

* tables
* columns
* relationships
* indexes
* modules

The JSON structure is already defined and stored in the codebase.

---

# 🖥️ FRONTEND REQUIREMENTS

Use:

* React (preferred)
* TypeScript
* Tailwind CSS
* Zustand or Redux (state management)
* React Flow (for diagram rendering)

---

# 🎨 CORE FEATURES

## 1. ER DIAGRAM CANVAS

* Use React Flow to render:

  * Nodes = Tables
  * Edges = Relationships

### Node Design:

* Table name (header)
* Column list:

  * Highlight PK (Primary Key)
  * Highlight FK (Foreign Key)
* Scrollable if too many columns

---

## 2. TABLE SELECTION PANEL

Left sidebar:

* List all tables
* Checkbox selection
* Group by modules (sales, inventory, etc.)

### Actions:

* Select all
* Deselect all
* Search table

---

## 3. RELATIONSHIP FILTERING

Add filters:

* Show only:

  * Direct relationships
  * One-to-many
  * Many-to-one
* Toggle:

  * Show orphan tables (no relation)
  * Show only connected graph

---

## 4. DYNAMIC GRAPH RENDERING

* When user selects tables:

  * Only render selected tables
  * Auto-filter relationships
* Hide unrelated nodes

---

## 5. AUTO LAYOUT ENGINE

Use:

* dagre or elkjs

Features:

* Auto arrange nodes
* Avoid overlapping edges
* Support:

  * hierarchical layout
  * force layout (optional)

---

## 6. INTERACTION FEATURES

### Zoom & Pan

* Mouse zoom
* Drag canvas

### Node Interaction

* Click node → show details panel
* Hover → highlight connected edges

---

## 7. DETAILS PANEL (RIGHT SIDEBAR)

On table click:

Show:

* Table name
* Columns (type, PK, FK)
* Indexes
* Relationships

---

## 8. SEARCH FUNCTIONALITY

Global search:

* Table name
* Column name

Highlight matched nodes

---

## 9. RELATIONSHIP VISUAL STYLE

* One-to-many → arrow
* Many-to-one → reverse arrow
* Color-coded edges:

  * FK → blue
  * Critical relations → red

---

## 10. PERFORMANCE (VERY IMPORTANT)

* Support 100+ tables
* Lazy rendering
* Virtualization for large graphs

---

# ⚙️ BACKEND (OPTIONAL)

* Cache schema
* Versioning support

# 🧠 GRAPH BUILDING LOGIC

* Convert JSON → nodes + edges

Node:
{
id: table_name,
data: { columns, metadata }
}

Edge:
{
source: from_table,
target: to_table,
label: relationship_type
}
